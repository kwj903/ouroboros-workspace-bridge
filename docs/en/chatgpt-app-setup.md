# Connect as a ChatGPT custom app

This guide explains what to enter in ChatGPT's **New app** screen after starting Ouroboros Workspace Bridge locally.

![ChatGPT new app setup screen](../project/chatgpt-new-app-setup.png)

## Before you start

Make sure the local session is running first.

```bash
uv sync
uv run woojae setup
uv run woojae start
```

During setup, choose the `WORKSPACE_ROOT` ChatGPT may access and the default help language. Existing shell environment values such as `WORKSPACE_ROOT`, `NGROK_HOST`, `MCP_ACCESS_TOKEN`, and `WOOJAE_HELP_LANG` are respected.

Use this command to copy the real MCP URL for connection:

```bash
uv run woojae copy-url
```

Use this command only when you want a redacted preview:

```bash
uv run woojae mcp-url
```

`mcp-url` may hide the token. Use the real URL copied by `copy-url` in the ChatGPT app setup screen.

## What to enter in each field

| Field | Value | Notes |
| --- | --- | --- |
| Icon | Optional | The app can be created without an icon. If you add one, use a PNG of at least 128x128. |
| Name | `Ouroboros Workspace Bridge` | This is the name shown in the ChatGPT app list. `Ouroboros Bridge` is also fine. |
| Description | `Local MCP bridge for reviewing file edits and commands before approval.` | Optional, but recommended so the app is easy to recognize later. |
| MCP server URL | URL copied by `uv run woojae copy-url` | Do not guess this URL. Paste the real URL generated by the local command. |
| Authentication | Prefer `No auth`, `None`, or the equivalent option | This bridge normally uses an `access_token` in the MCP URL query string. Do not choose OAuth unless you intentionally configured OAuth support. |
| Advanced OAuth settings | Usually leave empty | Do not change this unless your bridge is explicitly configured for OAuth. |
| Security warning checkbox | Check after reading the warning | The Create button may stay disabled until this is checked. |
| Create | Click after all required fields are filled | Refresh or reconnect the connector after creation. |

## MCP server URL

The MCP server URL is the most important field. It usually has this shape:

```text
https://<NGROK_HOST>/mcp?access_token=<TOKEN>
```

Treat the real token-bearing URL as a secret.

- Do not paste it in GitHub issues.
- Do not expose it in docs or screenshots.
- Do not share it with other people.
- Be careful with terminal output and clipboard history after connecting.

If `NGROK_HOST` is not configured, a temporary ngrok URL may be used. Temporary URLs can change after restart, which means the MCP server URL in the ChatGPT app may need to be updated.

## Authentication

The screenshot may show `OAuth` as the selected default. For the normal Workspace Bridge flow, use a **token-bearing MCP URL**, not OAuth.

Choose one of these options when available:

- `No auth`
- `None`
- a direct URL option without OAuth

If the UI only allows OAuth or requires OAuth settings, check whether you are in the correct custom MCP app creation flow. Filling arbitrary OAuth values can make the connection fail.

## Security warning checkbox

The warning explains that custom MCP servers can access data and tools. That warning is important.

Ouroboros Workspace Bridge is designed to avoid direct execution by using this flow:

```text
ChatGPT request
  -> Local MCP bridge
  -> Pending bundle
  -> localhost review UI approval
  -> File change or command
```

Only connect your own trusted local bridge, and approve only expected bundles in the review UI.

## After creating the app

Check the local session status:

```bash
uv run woojae status
```

Open the local review UI:

```text
http://127.0.0.1:8790/pending
```

For the first test, ask ChatGPT to perform a harmless check.

Example:

```text
Use this workspace directory: /path/to/your/project
Show me a brief overview of this directory's structure and tell me what kind of project it looks like.
```

When the expected command bundle appears in the review UI, inspect it and approve it.

## Common issues

### The Create button is disabled

- Make sure required fields such as name and MCP server URL are filled.
- Make sure the security warning checkbox is checked.
- Make sure the URL starts with `https://`.

### I do not know which URL to enter

Do not build the URL manually. Use:

```bash
uv run woojae copy-url
```

For a redacted preview only, use:

```bash
uv run woojae mcp-url
```

### I do not know which authentication option to choose

The normal bridge flow uses an `access_token` in the URL. Prefer `No auth`, `None`, or the equivalent direct URL option. Do not configure OAuth unless you intentionally added OAuth support.

### The app is created, but tools do not work

Check the local session:

```bash
uv run woojae status
```

Then confirm:

- `uv run woojae start` is still running.
- the ngrok URL has not changed.
- the ChatGPT app's MCP server URL has been updated to the current URL.
- `http://127.0.0.1:8790/pending` opens locally.
